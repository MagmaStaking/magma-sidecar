# Release verification

This guide is for operators who want to verify the Magma signing key, release
manifest, package hashes, and GitHub build provenance independently. The
standard APT installation already verifies signed repository metadata and the
selected package's SHA256 automatically.

## Signing key

The Magma APT and release-manifest signing key has fingerprint:

```
17E195DD0627D3422E8896F5EA0AA065F4C8C3E7
```

Verify the published key before trusting it:

```bash
(
  set -euo pipefail
  BASE="https://magmastaking.github.io/magma-sidecar-apt-repo"
  EXPECTED_FINGERPRINT="17E195DD0627D3422E8896F5EA0AA065F4C8C3E7"
  KEY_TMP="$(mktemp)"
  trap 'rm -f "${KEY_TMP}"' EXIT

  curl -fsSL "${BASE}/magma-apt-key.gpg.bin" -o "${KEY_TMP}"
  ACTUAL_FINGERPRINT="$(
    gpg --batch --show-keys --with-colons "${KEY_TMP}" \
      | awk -F: '$1 == "fpr" { print $10; exit }'
  )"

  if [ "${ACTUAL_FINGERPRINT}" != "${EXPECTED_FINGERPRINT}" ]; then
    echo "Magma signing-key fingerprint mismatch" >&2
    echo "expected: ${EXPECTED_FINGERPRINT}" >&2
    echo "actual:   ${ACTUAL_FINGERPRINT:-<missing>}" >&2
    exit 1
  fi

  echo "Verified Magma signing key: ${ACTUAL_FINGERPRINT}"
)
```

Confirm the expected fingerprint through an independent Magma communication
channel before relying on the downloaded key.

## Signed release manifest and package hashes

Each published version has a manifest containing its exact source commit,
multi-arch container digest, and SHA256 hashes for the `amd64` and `arm64`
Debian packages.

Replace `X.Y.Z`, then run:

```bash
(
  set -euo pipefail
  VERSION="X.Y.Z"
  BASE="https://magmastaking.github.io/magma-sidecar-apt-repo"
  RELEASE_BASE="https://github.com/MagmaStaking/magma-sidecar/releases/download/v${VERSION}"
  VERIFY_DIR="$(mktemp -d)"
  trap 'rm -rf "${VERIFY_DIR}"' EXIT
  cd "${VERIFY_DIR}"

  curl -fsSLO "${BASE}/magma-apt-key.gpg.bin"
  curl -fsSLO "${BASE}/releases/${VERSION}/release-manifest.txt"
  curl -fsSLO "${BASE}/releases/${VERSION}/release-manifest.txt.asc"
  curl -fsSLO "${RELEASE_BASE}/magma-sidecar_${VERSION}_amd64.deb"
  curl -fsSLO "${RELEASE_BASE}/magma-sidecar_${VERSION}_arm64.deb"

  mkdir -m 0700 gnupg
  gpg --batch --homedir "${VERIFY_DIR}/gnupg" \
    --import magma-apt-key.gpg.bin
  gpg --batch --homedir "${VERIFY_DIR}/gnupg" \
    --verify release-manifest.txt.asc release-manifest.txt
  sha256sum --check release-manifest.txt
)
```

The signature and both package checks must succeed.

## GitHub build provenance

The GitHub release packages also carry build provenance generated by the
release workflow. With GitHub CLI 2.49.0 or newer:

```bash
VERSION="X.Y.Z"
gh release download "v${VERSION}" \
  --repo MagmaStaking/magma-sidecar \
  --pattern '*.deb' \
  --dir /tmp/magma-sidecar-verify

for deb in /tmp/magma-sidecar-verify/*.deb; do
  gh attestation verify "${deb}" --repo MagmaStaking/magma-sidecar
done
```

The verified attestation must identify `MagmaStaking/magma-sidecar` as the
source repository and the expected release commit.
